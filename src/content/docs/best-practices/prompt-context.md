---
title: 提示词上下文设计
description: 如何组织上下文，让模型回答更容易审阅和复用。
---

好的上下文设计能让模型输出更容易评估。一个可用的提示词应清晰拆分任务、资料、约束和输出格式。

## 推荐结构

使用四个区块：

1. **任务：** 模型需要完成什么。
2. **上下文：** 模型可以依赖哪些事实。
3. **约束：** 风格、安全、范围和禁止假设。
4. **输出：** 格式、长度和必填字段。

## 示例

```text
任务：为技术支持工程师总结集成问题。
上下文：只使用下方记录。
约束：不要提及客户名称。不要猜测根因。
输出：返回症状、可能原因和下一步检查。
```

## 审阅清单

- 资料和指令已经分开。
- 输出格式明确。
- 提示词说明了哪些内容不能推断。
- 发送给模型前已移除敏感标识。
