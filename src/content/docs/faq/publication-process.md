文档通过审阅流程发布。

## 步骤

1. 作者创建草稿。
2. 在 GitHub 打开 pull request。
3. GitHub Actions 检查敏感词并构建站点。
4. 审阅者批准变更。
5. pull request 合并到 `main`。
6. GitHub Actions 将站点部署到 GitHub Pages。

## 为什么需要这个流程

审阅流程可以避免误发私有术语、客户细节、缺少依据的声明和损坏页面。
