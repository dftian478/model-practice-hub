如果回答格式在多次运行中变化，应明确期望输出结构，并在下游使用前校验结果。

## 检查项

1. 在提示词中加入具体 schema 或条目结构。
2. 如果格式不常见，加入一个短示例。
3. 移除互相冲突的指令。
4. 如 API 支持，降低随机性参数。
5. 接收输出前加入应用侧校验。

## 示例指令

```text
只返回三个部分：
1. 摘要
2. 依据
3. 下一步检查
```

## 何时升级处理

如果自动化流程强依赖格式一致性，不要只依赖提示词。应在应用层增加校验和重试逻辑。
